Le système de template : Introduction

Mélange de code PHP et de HTML

Le bout de code ci-dessous correspond à l'entête d'un fichier .html. Du code PHP a été mélangé au code HTML. On reconnaît le code PHP car il est encadré par "<?php" et "?>" :

	
<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Transitional//EN"
    "http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd">
<html xmlns="http://www.w3.org/1999/xhtml">

<head>
<!-- META-DONNEES  -->
<title><?php echo utf8_decode($domPageInfo->getAttribute('title'));  ?></title>
<meta name="Author" lang="fr" content="<?php echo utf8_decode($domPageInfo->getAttribute('auteur')); ?>"/>
<meta name="Description" content="<?php echo utf8_decode($domPageInfo->getAttribute('description')); ?>"/>
<meta name="Keywords" content="<?php echo utf8_decode($domPageInfo->getAttribute('motsclefs'));  ?>"/>
<meta http-equiv="content-type" content="text/html; charset=ISO-8859-1" />
<meta name="google-site-verification" content="hlTcDZ_jeAX3P2jrurWiFljMitDPFq2uWCdB2mHtGuY" />

Bref, ça peut vite devenir le bazar, surtout si le code PHP doit afficher du code HTML. Une alternative à ce genre de pratique est l'utilisation d'un système de template.

Qu'est ce qu'un système de template en PHP ? Il s'agit d'un moyen permettant à un programmeur PHP de travailler de son côté, tout en permettant à un graphiste HTML par exemple de travailler du sien. Le programmeur PHP aura un fichier .php, le graphiste un fichier .html (template). Le système de template mettra en relation ces deux fichiers afin de n'en former plus qu'un seul à la fin. L'intéret d'un tel système est justement de ne pas mélanger PHP et HTML, pour faciliter la vie des graphistes en général. Le système de template qui permet de gérer les thèmes sans jamais aller modifier les fichiers PHP (Source: www.developpez.com).

Pour changer de thème il suffit simplement de changer de template.

Les balises ("tags")

Le code ci-dessous correspond au code d'au-dessus, mis à part que le code PHP a été élégamment remplacé par des balises (tags) du type : "{NOM_DE_LA_BALISE}" :

	
<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Transitional//EN"
    "http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd">
<html xmlns="http://www.w3.org/1999/xhtml">

<head>
<!-- META-DONNEES  -->
<title>{TITRE}</title>
<meta name="Author" lang="fr" content="{AUTEUR}"/>
<meta name="Description" content="{DESCRIPTION}"/>
<meta name="Keywords" content="{MOTS_CLEFS}"/>
<meta http-equiv="content-type" content="text/html; charset=ISO-8859-1" />
<meta name="google-site-verification" content="hlTcDZ_jeAX3P2jrurWiFljMitDPFq2uWCdB2mHtGuY" />

Le serveur va lire le fichier .html, chercher si il y a des balises "{}" et remplacer leur contenu par les données correspondante. Par example, la balise {AUTEUR} sera remplacée par Jacques Chirac.

Cela veut dire que c'est le graphiste qui contrôle l'information en ajoutant ou supprimant judicieusement les balises "{}". Si dans le fichier .html vous ne voulez pas afficher le titre, il suffit simplement de supprimer la balise {TITRE}.
